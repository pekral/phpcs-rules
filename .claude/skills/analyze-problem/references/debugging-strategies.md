# Debugging Strategies

## Divide and Conquer
- Narrow down where the issue occurs
- Split system into parts

## Reproduce the Problem
- Always try to reproduce reliably
- If not reproducible, focus on logging

## Isolate Variables
- Change one thing at a time

## Binary Search Debugging
- Remove half of the system to locate issue faster

## Compare with Known Good State
- Compare with working version

## Logs and Traces
- Inspect logs first

## Environment Differences
- Compare local vs production
